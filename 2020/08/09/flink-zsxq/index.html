介绍


进知识星球的小伙伴有的是刚接触 Flink 的，有的是根本没接触过的，有的是已经用 Flink 很久的，所以很难适合所有的口味。


我一向认为对一门技术的学习方式应该是：

  • 了解（知道它的相关介绍、用处）
  • 用（了解常用 API）
  • 用熟（对常用 API 能够用熟来，并了解一些高级 API）
  • 解决问题（根据业务场景遇到的问题能够定位问题并解决）
  • 看源码（深入源码的实现，此种情况主要是兴趣爱好驱动）
